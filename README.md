# 🌿 PlantScan — Приложение для обнаружения болезней растений

PlantScan — это мобильное приложение для определения болезней растений с использованием свёрточных нейронных сетей и снимков с дронов. Приложение разработано на Flutter, использует YOLOv8 для обнаружения объектов и Google Gemini AI для интеллектуальных рекомендаций по лечению.


## ✨ Возможности

- **Обнаружение болезней растений**: Загружайте или фотографируйте растения для выявления болезней с помощью обученной модели YOLOv8
- **Анализ снимков с дронов**: Поддержка изображений с дронов для мониторинга полей большого масштаба
- **Рекомендации ИИ**: Получайте рекомендации по лечению от Google Gemini AI
- **История болезней**: Отслеживайте все предыдущие сканирования и результаты
- **Классификация болезней листьев**: Определение ржавчины, парши, мучнистой росы и других заболеваний
- **Мультиязычность**: Поддержка английского и русского языков
- **Кроссплатформенность**: Работает на Android и iOS

## 🛠 Технологический стек

- **Фреймворк**: Flutter 3.22.2
- **Язык**: Dart
- **Управление состоянием**: Provider + ChangeNotifier
- **Обнаружение объектов**: YOLOv8 (через собственный API)
- **ИИ-помощник**: Google Gemini AI API
- **Камера**: phone_camera
- **Локальное хранилище**: SharedPreferences
- **HTTP-клиент**: http
- **Архитектура**: Model-View-ViewModel (MVVM)
- **Бэкенд API**: Flask (для сервинга модели YOLOv8)

## 🔄 Поток данных

Приложение следует паттерну MVVM с Provider для управления состоянием:

1. **Пользователь** делает или выбирает фото через Камеру или Галерею
2. **PhotoService** обрабатывает изображение для загрузки
3. **ApiService** отправляет изображение на Flask-бэкенд с YOLOv8
4. **Бэкенд** возвращает предсказания болезней с оценкой уверенности
5. **ResultScreen** отображает результаты обнаружения
6. **GeminiService** предоставляет рекомендации по лечению на основе ИИ
7. **История болезней** сохраняется локально через SharedPreferences

## 🚀 Начало работы

### Предварительные требования

- Flutter SDK 3.22.2 или выше
- Dart 3.4.3 или выше
- Android Studio или VS Code
- Python 3.9+ (для бэкенд API)
- Ключ Google Gemini API

### Установка

1. Клонируйте репозиторий

```
git clone https://github.com/Mamblz/PlantScan.git
cd PlantScan
```

2. Установите зависимости Flutter

```
flutter pub get
```

3. Настройте бэкенд API

Приложению требуется Flask-сервер с моделью YOLOv8. Клонируйте и настройте репозиторий бэкенда:

```
git clone https://github.com/Mamblz/PlantsAPI.git
cd PlantsAPI
pip install -r requirements.txt
python app.py
```

4. Настройте эндпоинты API

Обновите базовый URL в файле `lib/services/api_service.dart`, указав адрес вашего сервера:

```dart
static const String baseUrl = 'http://IP_ВАШЕГО_СЕРВЕРА:5000';
```

5. Настройте Google Gemini API

Получите API-ключ в Google AI Studio и добавьте его в `lib/services/gemini_service.dart`:

```dart
static const String apiKey = 'ВАШ_GEMINI_API_КЛЮЧ';
```

6. Запустите приложение

```
flutter run
```

## 📡 Бэкенд API

Приложение взаимодействует с бэкендом на Flask, который обслуживает модель YOLOv8, обученную на наборе данных болезней растений.

### Эндпоинты API

| Метод | Эндпоинт | Описание |
|--------|----------|----------|
| POST | /predict | Загрузка изображения для предсказания болезни |
| GET | /health | Проверка работоспособности API |

### Формат запроса

```
POST /predict
Content-Type: multipart/form-data

image: [файл]
```

### Формат ответа

```json
{
  "predictions": [
    {
      "class": "rust",
      "confidence": 0.95,
      "bbox": [x1, y1, x2, y2]
    }
  ],
  "annotated_image": "base64_закодированное_изображение"
}
```

## 🎯 Поддерживаемые болезни

Модель YOLOv8 способна обнаруживать следующие болезни растений:

| Болезнь | Описание |
|---------|----------|
| Ржавчина яблони | Грибковое заболевание, вызывающее оранжевые пятна на листьях |
| Парша яблони | Грибковое заболевание, вызывающее тёмные поражения на листьях и плодах |
| Мучнистая роса | Белый мучнистый грибковый налёт на листьях |
| Здоровое растение | Болезней не обнаружено |

## ⚠️ Важные замечания

- Для работы обнаружения болезней бэкенд API должен быть запущен
